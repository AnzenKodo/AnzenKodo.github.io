Spam

Any kind of unwanted, unsolicited digital communication that gets sent out in bulk.

Spam is sent via email, text messages, phone calls, or social media.

Types of spam
  • Phishing emails
    • Type of spam cybercriminals sent to many people, hoping to "hook" a few people.
    • Phishing email trick victims into giving up sensitive information.
  • Email Spoofing
    • Spoofed emails mimic, or spoof, an email from a legitimate sender, and ask you to take some sort of action.

Virus

Virus is a malware that relies on someone or something to propagate from one system to another.

Phases of Virus

Trojan horse

Programs that performs malicious some harmless activity in addition to some malicious activities.

Examples:

  • A game could do malicious while the victim is playing.
  • Downloading cracked application.
  • Opening infected attachment.

Trapdoor

Trapdoor or a backdoor is a vulnerability found in the system where attacker can planned his malicious code and make a secret unauthorized entry to the system.

Logic bomb

Only attacks when Boolean logic(true or false) is satisfied.

It has two phases:

  1. Payload: Malicious code is placed.
  2. Trigger: Boolean value is satisfied or the condition is become true.

IPSec

It is a protocol which resides on the network layer.

It is an Internet Engineering Task Force (IETF) standard suite of protocols between two communication points across the IP network.

It defines the encrypted, decrypted, and authenticated packets.

IPSec Packet

IPSec 3 function areas

Use of IPSec

  • To encrypt application layer data.
  • To provide security for routers.
  • Sending the data across in public network.
  • To provide authentication without encryption that the data originated from a known sender.

Components of IPSec

IPSec Tunnel Mode vs Transport Mode

Tunnel Mode

In tunnel mode, the entire original IP packet is encapsulated to become the payload of a new IP packet.

Transport Mode

The main difference in transport mode is that it retains the original IP header.

In transport mode, encrypted traffic is sent directly between two hosts that previously established a secure IPsec tunnel.

Network-attached storage (NAS) security is the measures a company takes to protect critical enterprise and customer data within NAS environments from both internal and external threats.

Rather than a specific, vendor-offered type of software,¬†NAS¬†security is a set of best practices that businesses implement to protect their networked storage.

Top 4 NAS vulnerabilities

Unsecure network configurations

If the network to which the NAS storage devices are connected is an open network, all NAS devices are automatically vulnerable to any threats on that public network

Unprotected passwords

Network access passwords, storage management passwords, and any other credentials left in the open or shared unsafely are at risk of wrongful use.

Insufficient authentication

If users on the network aren‚Äôt required to prove their identity and their right to access the system, employees untrained in security practices could wreak havoc on a NAS.

The internet

Because some NAS systems are connected to the public internet rather than only a company network, they‚Äôre exposed to the internet‚Äôs vulnerabilities, too

Public internet vulnerabilities include denial of service (DoS) attacks, botnets, and malware-infected web pages.

Key components of NAS security

Access controls

All users on the network should be required to authenticate their identity at each entry point.

Example, knowing a password, having a generated passcode, or using biometrics like facial recognition.

Device approval

If a NAS system is connected to a company network, all other non-storage devices that connect to the network, like employee mobile devices, should require approval.

Network security

Protecting the network where NAS systems reside is critical to protecting the data.

Internet security practices

Some NAS systems are connected to the internet and must be protected from internet threats, like website malware and botnets.

Regular system and software updates

NAS management software should be updated immediately when a new update is released.

NAC

Network Access Control is a centralized solution to end-point security that focuses on network visibility and strict access management by enforcing policies across all users and devices.

The objective of Network Access Control is to block unauthorized users or devices from entering a private corporate network.

Use cases for Network Access Control

  • IoT
    • The right NAC solution will identify and monitor IoT devices, in addition to traditional devices.
  • BYOD (Bring Your Own Device)
    • With employees working remotely from personal computers or accessing the corporate network from personal phones, a proper NAC solution must also be able to handle permissions and authentication of unfamiliar devices attempting to access the network.
  • Incident Response
    • In addition to simply controlling network access, a robust network access control solution should be able to respond to threats quickly and effectively.

  • Medicine

    • In the world of healthcare, there is a growing reliance on the Internet of Medical Things (IoMT) devices.

    • Properly structured NAC solutions can provide the necessary protection of sensitive personal data and medical records in a network with multiple users and IoMT devices.

      SNMP

SNMP stands for Simple Network Management Protocol.

It is basically an Internet Standard Protocol which is used for monitoring and organizing information about the devices on IP network by sending and receiving requests.

This protocol is used for organizing information from devices like switches, modems, routers, servers, printers etc.

Currently, there are 3 versions of SNMP ‚Äì SNMPv1, SNMPv2, SNMPv3

SNMP components

SNMP Manager

It is a centralized system used to monitor network. It is also known as Network Management Station (NMS)

SNMP agent

It is a software management software module installed on a managed device. Managed devices can be network devices like PC, routers, switches, servers, etc.

Management Information Base

MIB consists of information on resources that are to be managed.

This information is organized hierarchically.

It consists of objects instances which are essentially variables.

IDS

An Intrusion Detection System (IDS) is a system that monitors network traffic for suspicious activity and issues alerts when such activity is discovered.

It is a software application that scans a network or system for harmful activity or policy breaching.

Any malicious venture or violation is normally reported either to an administrator or collected centrally using a Security Information and Event Management (SIEM) system.

Classification of intrusion detection system:

Network Intrusion Detection System (NIDS)

Set up at a planned point within the network to examine traffic from all devices on the network.

It performs an observation to examine traffic from all devices on the network.

Host Intrusion Detection System (HIDS)

Run on independent hosts or devices on the network.

Monitors the incoming and outgoing packets from the device only and will alert the administrator if malicious activity is detected.

It takes a snapshot of existing system files and compares it with the previous snapshot.

Protocol-based Intrusion Detection System (PIDS)

Comprises a system or agent that would consistently resides at the front end of a server, controlling and interpreting the protocol between a user/device and the server.

It is trying to secure the web server by regularly monitoring the HTTPS protocol stream and accept the related HTTP protocol.

Application Protocol-based Intrusion Detection System

It is a system or agent that generally resides within a group of servers.

It identifies the intrusion by monitoring and interpreting the communication on application-specific protocols.

Hybrid Intrusion Detection System

It is made by combination of two or more approaches of the intrusion detection system.

Host agent or system data is combined with network information to develop a complete view of the network system.

It is more effective in comparison to the other IDS.

Detection Method of IDS

Signature based

Detects the attacks on the bases of the specific patterns such as number of bytes or number of 1's or number of 0's in the network traffic.

It detects on basis of the already known malicious instruction sequence that is used by the malware.

The detected patterns are known as signatures.

Anomaly based

It detect unknown malware as new malware are developed rapidly.

Machine Learning is used to develop IDS.

Comparison of IDS with Firewalls

Firewall restrict access between networks to prevent intrusion and if an attack is from inside the network it doesn't signal.

An IDS describes a suspected intrusion once it has happened and then signals an alarm.

Firewall

A firewall is a network security device that monitors incoming and outgoing network traffic and permits or blocks data packets based on a set of security rules.

It purpose is to establish a barrier between your internal network and incoming traffic from external source like internet. In order to block malicious traffic like viruses or hacker.

Firewall Characteristics

Types of Firewall

Packet Filtering

The packet filtering firewall analyses for the source and destination IP address, source & destination port number and protocol IDs of IP packets as per an Access Control List (ACL)

It create a checkpoint at a traffic route.

It is oldest type of firewall.

Application Proxy

Application proxy is a program running on the firewall the emulates both ends of a network connections.

Computer communicates with each other by passing all network traffic through the proxy program.

The proxy program evaluates the data sent from client and decides which to pass on and which to drop.

Circuit Gateway

This can be a stand-alone system or it can be the specialized function performed by an application level gateway for certain applications.

It does not permit end-to-end TCP connection. Rather, the gateway sets two TCP connections.

Firewall Design Principles

Developing Security Policy

It is a very essential part of firewall design.

It is designed according to requirement of the company or client to know which kind of traffic is allowed to pass.

Without a proper security poly, it is impossible to restrict or allow a specific user or worker in a compony network or anywhere else.

Simple Solution Design

If the design of the solution is complex, then it will be difficult to implement it.

If the solution is easy, then it will be easier to easier to implement it.

A simple design is easier to maintain.

We can make upgrades in simple design according to the new possible treats leaving it with an efficient but more simple structure.

Choosing the Right Device

If we use the wrong device for the wrong problem, the network becomes vulnerable.

If the outdated device is used for a designing firewall, it exposes the network to risk and is almost useless.

Layered Defense

A network defense must be multiple layered in the modern world because if the security is broken, the network will be exposed to external attacks.

Consider Internal Threats

While giving a lot of attention to safeguarding the network or device from external attacks.

The security becomes weak in case of internal attacks and most of the attacks are done internally as it is easy to access and designed weakly.
