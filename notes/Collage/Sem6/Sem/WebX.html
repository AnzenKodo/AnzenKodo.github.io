1. Explain Semantic Web Stack

The Semantic Web is an extension of the World Wide Web that aims to make information on the web more understandable to computers by adding a layer of meaning to it.

  • Identifiers
    • Uniform Resource Identifier (URI): A string that acts as a unique identifier for a web resource. URIs can be used to identify documents, images, people, or anything else on the web.
    • Character Set: Unicode: A character encoding standard that allows for the representation of text in most of the world's writing systems.
  • Syntax
    • Extensible Markup Language (XML): XML provides a way to structure data and define its meaning.
  • Data Interchange
    • Resource Description Framework (RDF): A framework for representing information about resources in the form of triples.
    • Eg: ("Paris", "is located in", "France")
  • Taxonomies
    • RDF Schema (RDFS): A vocabulary for describing properties and classes of resources. RDF.
  • Ontologies
    • Web Ontology Language (OWL): OWL allows you to define complex relationships between classes, properties, and individuals.
  • Rules
    • RIF/SWRL : Rule Interchange Format (RIF) and Semantic Web Rule Language (SWRL) are languages for expressing rules and inference in the Semantic Web.
  • Proof
    • It refers to the mechanisms that can be used to verify the truth or accuracy of information on the Semantic Web.
  • Trust
    • It refers to the mechanisms that can be used to establish trust between different parties on the Semantic Web.
  • Querying
    • SPARQL Protocol and RDF Query Language (SPARQL): A query language for retrieving and manipulating data stored in RDF format.
  • User Interface and Applications
    • This layer refers to the applications that users interact with to access and use the Semantic Web.

2. Explain Arrow Function in TypeScript

Arrow functions in TypeScript are a concise way to write anonymous function expressions.

They were introduced in ES6 and provide a more structured and readable way to create functions.

 TypeScript, arrow functions use the => syntax, eliminating the need for the function keyword and allowing for shorter function definitions.

Syntax:

.ts 
(parameters...) => { }

Example:

  • Arrow function with parameters

    .ts 
    const add = (a:number, b:number) => {
	console.log(a + b);
}
add(1, 2);
// Output: 3

  • Arrow function without parameters

    .ts 
    const hello = () => console.log("Hello LU")
hello();
// Output: Hello LU

3. What is Routing in AngularJS

Routing in AngularJS is a core feature that allows developers to create Single Page Applications (SPAs) with multiple views. It enables the application to load different parts of the application without reloading the entire page.

  • Include the angular-route.js library in your HTML file.
    • <script src="angular-route.js"></script>
  • Configure the routes using the $routeProvider service in the config() method of your module. For each route, specify the URL and the corresponding template or controller.

Example

.html 
<head>
	<script src="ajax.googleapis.com/anguler.min.js"></script>
</head>
<body>
	<div ng-app="app">
		<a href="#/">Home</a>
		<a href="#/about">About</a>
	</div>
	<script>
		angular.module('app', ['ngRoute'])
		.config(($routeProvider) => {
			$routeProvider
			.when('/', { templateUrl: 'home.html' })
			.when('/about', { templateUrl: 'about.html' })
		});
	</script>
</body>

4. Default database in MongoDB: Local, Admin & Config

In MongoDB, there are three special databases that are created by default: local, admin, and config. These databases have specific purposes and shouldn't be directly modified during regular use.

Here's a breakdown of each:

  1. Local Database:
    • This database stores internal information about the MongoDB server itself.
    • You generally wouldn't interact with the local database directly.
  2. Admin Database:
    • This database is crucial for administrative tasks like user management and configuration.
    • It stores information about users, roles, and authentication mechanisms.
  3. Config Database (Sharded Cluster Only):
    • It stores configuration information for the sharded cluster, such as chunk sizes and balancer settings.

Example how to access default databases:

.sh 
> use local  # To access local database
> use admin  # To access admin database
> use config # To access config database

  • JavaScript sends asynchronous request: The JavaScript code sends the asynchronous HTTP request to the web server.
  • Web server retrieves data: The server receives the request and fetches the necessary data.
  • Server sends response: The server then sends the response back to the browser.
  • Browser receives response: The browser receives the response from the server.
  • JavaScript processes response: The JavaScript code then extracts the data from the response and uses it to update the web page content.
  • Browser updates content: The browser updates the webpage with the new information, without reloading the entire page.

14. What is flask URL Building?

Flask URL building is a crucial feature in Flask that allows developers to dynamically generate URLs for specific functions within a Flask web application.

Flask URL building involves using the url_for() function to dynamically construct URLs for specific functions within a Flask application.

There are two types of URL in flask:

  • Static URL: The static url don't change and remain static.
    • The url_for() don't require parameters for static functions:
    • Syntax: url_for(url)

  • Dynamic URL: The dynamic url can be changed with provided parameters

    • Syntax url_for(url, parameter...)

      .python 
      from flask import *

app = Flask(__name__)

@app.route('/admin') # Supports Static url
def admin():
	return 'Hello Admin'

@app.route('/guest/<id>') # Supports dynamic url
def guest(guest):
	return 'Hello Guest'

@app.route('/user/<name>')
def index(name):
	if name == 'admin':
		return redirect(url_for('admin'))
	else:
		return redirect(url_for('guest', id=name))

if __name__ == '__main__':
	app.run()

15. AngularJS Data Binding

AngularJS data binding is a powerful feature that simplifies the process of creating dynamic and responsive web applications.

It automatically synchronizes data between the model (application data) and the view (HTML template) in your AngularJS application.

Types of Data Binding in AngularJS:

  1. One-Way Data Binding (Default):

    • In this type of binding, changes made to the model are reflected in the view, but changes made in the view do not update the model. This is the default behavior in AngularJS.
    • You use double curly braces {{ expression }} within your HTML template to display model data.

    • Example:

      .html 
      <head>
	<script src="ajax.googleapis.com/anguler.min.js"></script>
</head>
<body ng-app="" ng-init="name='LU'">
	<p>Welcome to {{ name }}</p>
</body>

  2. Two-Way Data Binding:

    • Two-way data binding allows changes made in both the model and the view to be automatically reflected in each other. This is achieved using the ng-model directive.
    • ng-model creates a two-way binding between an input element in the view and a property in the model.

    • Example:

      .html 
      <head>
	<script src="ajax.googleapis.com/anguler.min.js"></script>
</head>
<body ng-app="">
	<input type="text" ng-model="name" />
	<p>{{ name }}</p>
</body>

16. Inheritance in TS with Example

Inheritance in TypeScript allows a class to inherit properties and methods from another class, enabling code reuse and extensibility.

TypeScript supports both single-level and multi-level inheritance.

Single-level Inheritance

In single-level inheritance, a child class extends a single parent class.

.ts 
class Collage {
	print_collage():void {
		console.log("Study in LU");
	}
}

class Student extends Collage {
	constructor(name: string) {
		this.name = name;
	}
	print_name():void {
		console.log("Name: "+name);
	}
}

const std = new Student("Jonney Bins");
std.print_name();
// Name: Jonney Bins
std.print_collage();
// Study in LU

Multi-level Inheritance

Multi-level inheritance involves a child class inheriting from a parent class, which itself inherits from a grandparent class.

.ts 
class Trusty {
	print_trusty():void {
		console.log("Trusty: Monjolika");
	}
}

class Collage extends Trusty {
	print_collage():void {
		console.log("Study in LU");
	}
}

class Student extends Collage {
	constructor(name: string) {
		this.name = name;
	}
	print_name():void {
		console.log("Name: "+name);
	}
}

const std = new Student("Jonney Bins");
std.print_name();
// Name: Jonney Bins
std.print_collage();
// Study in LU
std.print_trusty();
// Trusty: Monjolika

17. AngularJS Custom directives with examples.

AngularJS custom directives are a powerful feature that allows developers to extend the functionality of HTML elements.

They provide a way to create reusable components, add specific behaviors to existing elements, and enhance the overall modularity and maintainability of an AngularJS application.

The restrict property in a custom AngularJS directive defines where and how the directive can be used within your HTML templates.

Values for restrict:

  • 'E' (Element): It specifies that the directive can be used as a custom HTML element.
  • 'A' (Attribute): This allows the directive to be used as an attribute on existing HTML elements.
  • 'C' (Class): This lets you use the directive as a CSS class on an element.

  • Combine Values: You can also combine the values of restrict like AE, EC & AC.

    .html 
    <head>
	<script src="ajax.googleapis.com/anguler.min.js"></script>
</head>
<body>
	<div ng-app="app" ng-init="name='LU'">
		<collage name="LU"></collage>
	</div>
	<script>
		angular.module('app', []).directive('name', () => {
		    return {
		      restrict: 'E',
		      template: '<h1>Welcome to {{ name }}</h1>',
		    };
		});
	</script>
</body>

Output:

Welcome to LU

20. Explain controllers and scope in AngularJS with example.

Controllers in AngularJS:

  • Definition: Controllers in AngularJS control the data of AngularJS applications. They are regular JavaScript objects created by standard JavaScript object constructors.
  • Functionality: Controllers define the application behavior, handle user input, and interact with services to provide dynamic behavior to the application.
  • Syntax: Controllers are defined using the ng-controller directive in the HTML, specifying the controller name and its associated functionality.

Scope in AngularJS:

  • Definition: Scope in AngularJS is the binding part of the HTML view and JavaScript controller. It contains model data and connects the controller with views.
  • Functionality: Scope provides access to properties and methods defined in the controller, allowing data to be shared between the controller and the view.

  • Syntax: Scope is accessed using the $scope object in controllers, and changes made to the scope in the controller are reflected in the view.

    .html 
    <head>
	<script src="ajax.googleapis.com/anguler.min.js"></script>
</head>
<body>
	<div ng-app="app" ng-controller="control">
		<p>Welcome to {{ name }}</p>
	</div>
	<script>
		angular.module('app', []).controller('control', ($scope) => {
			$scope.name = "LU";
		}
	</script>
</body>

23. Explain routing using ng-route, ng-repeat, ng-style, ng-view with example.

24. Define RIA & Explain Characteristics of RIA

A Rich Internet Application (RIA) is a web application that provides a user experience similar to traditional desktop applications, with features such as offline use, advanced communications, and direct interaction with page elements12. RIAs incorporate additional technologies like real-time streaming, high-performance client-side virtual machines, and local caching mechanisms to enhance performance and responsiveness2.

Characteristics of Rich Internet Applications (RIA)

  1. Performance: RIAs can often perform better than traditional applications by offloading processing work to the client system.
  2. Offline use: RIAs allow users to work with the application even when connectivity is unavailable.
  3. Consistency of look and feel: RIA tools enable more careful control over the user interface and experience across different browsers, devices, and operating systems.
  4. Advanced Communications: Sophisticated communications with supporting servers through optimized network protocols can considerably enhance the user experience2.
  5. Rapid Development: RIA frameworks facilitate rapid development of rich user experiences through easy-to-use interfaces that help developers2.
  6. Direct Interaction: RIAs allow users to interact directly with page elements through editing or drag-and-drop tools, enhancing efficiency and user experience2.
  7. Improved Features: RIAs incorporate complex application screens with various mixed media, including different fonts, vector graphics, bitmap files, and online conferencing, using modern development tools2.
  8. Partial-page updating: RIAs can update specific parts of a page without reloading the entire page, reducing latency and increasing responsiveness2.

26. Explain MongoDB CRUD Operations with Example

MongoDB uses CRUD operations to Create, Read, Update, and Delete data within collections (similar to tables in relational databases).

1. Create (Insert):

  • Method:
    • db.collection.insertOne(document) (inserts a single document)
    • db.collection.insertMany(documents) (inserts multiple documents)

  • Example (Inserting a product):

    .javascript 
    db.products.insertOne({ name: "Jonney Bins" })

2. Read (Find):

  • Method:
    • db.collection.find(query) (finds documents based on a query)

  • Example:

    .js 
    db.products.find({ collage: { name: "LU" } }) // $lt is less than operator

3. Update:

  • Method:
    • db.collection.updateOne(filter, update) (updates a single document)
    • db.collection.updateMany(filter, update) (updates multiple documents)

  • Example (Updating product stock):

    .js 
    db.products.updateOne({ _id: 69 }, { name: "LU" })

4. Delete:

38. Explain Sharding

Sharding, in the context of databases, is a technique for distributing data across multiple servers (shards) to improve performance and scalability.

How sharding works:

  • Data Partitioning: The first step is to split the database into smaller, logical chunks called shards. This partitioning can be done based on various factors, such as:
    • Horizontal Partitioning: Dividing data based on a specific column or set of columns (e.g., sharding user data by user ID or geographical location).
    • Vertical Partitioning: Less common, but involves separating tables based on logical groupings (e.g., storing customer information in one shard and order history in another).
  • Shard Key Selection: A shard key is a crucial element used to determine which shard a particular piece of data belongs to. It's typically a column or combination of columns used for partitioning.
  • Distributed Storage: Each shard is then stored on a separate database server or node. This distribution allows for parallel processing of queries and improves overall database performance.

Benefits of Sharding:

  • Improved Performance: By distributing data across multiple servers, sharding reduces the load on any single server,.
  • Scalability: As your data volume grows, you can easily add more shard servers to handle the increased load.
  • Increased Availability: If a single shard server fails, only the data on that shard is affected.

Drawbacks of Sharding:

  • Increased Complexity: Sharding introduces additional complexity in managing data across multiple servers.
  • Limited Schema Flexibility: Changing the sharding scheme after implementation can be challenging.

* N-Triple & Turtle

N-Triples and Turtle are both serialization formats for representing RDF (Resource Description Framework) data34

N-Triples

N-Triples is a line-based, plain text format for encoding an RDF graph4. 

Each line represents a single RDF triple, consisting of a subject, predicate, and object, separated by whitespace and terminated by a period.

N-Triples uses IRIs (Internationalized Resource Identifiers) or blank nodes for subjects and objects.

Example:

<http://example.org/subject1>
<http://example.org/predicate1>
<http://example.org/object1>
<http://example.org/subject2>
<http://example.org/predicate2>
"object value" .

Turtle (Terse RDF Triple Language)

Turtle is a more compact and readable serialization format for RDF34. 

It allows writing down an RDF graph in a concise textual form.

Turtle supports various features such as:

  • Prefixed names for IRIs using @prefix
  • Shorthand notation for predicates using ; and ,
  • Blank nodes using [] or () for collections
  • Literals with language tags and data types

Example:

@prefix foaf: <http://xmlns.com/foaf/0.1/>
<http://example.org/alice> 
	foaf:name "Alice" ;
	foaf:knows <http://example.org/bob>
<http://example.org/bob> 
	foaf:name "Bob" .
