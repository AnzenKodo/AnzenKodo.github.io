EDA and it's types

Explanatory Data Analysis (EDA) in machine learning is a way of visualizing, summarizing and interpreting the information that is hidden in row and column format.

Once EDA is complete and insights are draw, its features can be used for supervised and unsupervised Machine Learning modeling.

Purpose of EDA

  • It helps to look at data before making any assumption.
  • Understanding of data can be easy with EDA.

Types of EDA

  • Univariate Non-graphical:
    • Analyzing individual variables without visualizations, using techniques like descriptive statistics and frequency tables.
    • Example:
      • - 3, 6, 8, 8, 9, 10, 13, 15, 16 - mean = 9.78 - median = 9 - mode = 8 -
  • Univariate graphical: Visualizing individual variables using charts like histograms, bar charts, and box plots to understand distribution, center, and spread.

    • Example: MANOVA

Anova

  • Stands for Analysis of Variance
  • It enables us to test significance of difference among more then two sample means.
  • Test statistics for ANOVA is F-test
  • Types of ANOVA:
    • One Way ANOVA
      • One factor of independent variable
      • Compares three or more levels of one factor.
    • Two Way ANOVA
      • Extension of One-way ANOVA
      • Mone then one factor or independent variable
  • Assumption for ANOVA
    • Samples follow normal distribution
    • Samples have been selected randomly and independently.
    • Each group should have common variance
    • Data are independent.

Machine learning

Machine learning is an application of AI through which the system can learn and improve by itself from experience.

Machine learning is training data into infromation.

Flowchart of ML:

Types of Machine Learning:

  1. Supervised Learning: Algorithms are trained with labeled data, where the input and output are known. This type is further divided into two categories:
    • Classification: Predicting categorical target variables, such as spam detection or disease diagnosis.
    • Regression: Predicting continuous target variables, such as stock prices or weather forecasting.
  2. Unsupervised Learning: Algorithms are trained with unlabeled data, where the input is known, but the output is not. This type is further divided into two categories:
    • Clustering: Grouping similar data points together, such as customer segmentation or anomaly detection.
    • Association: Identifying relationships between data points, such as market basket analysis or recommendation systems.
  3. Semi-Supervised Learning: Algorithms are trained with a combination of labeled and unlabeled data. This type is useful when obtaining labeled data is difficult or expensive.
  4. Reinforcement Learning: Algorithms learn by interacting with an environment and receiving feedback in the form of rewards or penalties. This type is often used in robotics, gaming, or navigation.

How ML application is developed

  1. Input Data: Gathering and preparing the data needed for training the machine learning model.
  2. Analyze Data: Examining the data to understand its structure, quality, and content.
  3. Find Patterns: Identifying patterns and relationships within the data using statistical or machine learning techniques.
  4. Prediction: Developing a machine learning model that can make predictions based on the identified patterns.
  5. Learn from Feedback: Improving the model's accuracy by providing feedback on its predictions and adjusting the model's parameters accordingly.

Application of ML

  1. Image Recognition: Identifying and detecting objects, people, or places in digital images.
  2. Speech Recognition: Converting voice instructions into text, also known as speech-to-text.
  3. Traffic Prediction: Predicting traffic conditions using information from users and Google Maps.
  4. Product Recommendations: Using machine learning to suggest products based on user behavior and preferences.
  5. Self-Driving Cars: Using machine learning to train cars to detect people and objects while driving, such as Tesla's self-driving cars.
  6. Email Spam and Malware Filtering: Automatically filtering emails as important, normal, or spam based on machine learning algorithms.
  7. Online Fraud Detection: Detecting fraudulent transactions by analyzing patterns in genuine transactions.
  8. Stock Market Trading: Using machine learning algorithms to predict stock market trends and make informed trading decisions.
  9. Medical Diagnosis: Using machine learning in medical science for disease diagnosis and building 3D models to predict the exact position of lesions.
  10. Automatic Language Translation: Converting text into known languages using machine learning algorithms, like Google's GNMT.

Lifecycle of DS

  • Business Understanding: Figuring out the problem and why it's important. This involves planning resources, assessing risks, and meeting stakeholders' expectations.
  • Data Collection: Getting data from different sources and making sure it's good quality. This can involve surveys, interviews, social media, web scraping, and analytics.
  • Data Preparation: Making sense of raw data by organizing, cleaning, and formatting it. This includes fixing errors, removing duplicates, and getting it ready for analysis.
  • Exploratory Data Analysis (EDA): Searching for patterns and trends in the data. This helps spot insights that can solve the problem at hand.
  • Modeling: Creating visual representations of data patterns using tools like decision trees and neural networks. This helps test ideas and relationships in the data.
  • Model Evaluation: Checking and finalizing the model to ensure it's accurate and useful. This step ensures the model is reliable for decision-making.
  • Model Deployment: Sharing the results with stakeholders for decision-making. The final model is a tool they can use to make informed choices based on the data analysis.
