    Features Of MongoDB

    MongoDB is an open source, document-oriented database designed with both scalability and developer agility in mind.

    Instead of storing your data in tables and rows as you would with a relational database, in MongoDB you store JSON-like documents with dynamic schemas(schema-free, schema less).

    • Document-Oriented Storage: MongoDB stores data as documents in collections, offering a flexible schema where documents within a collection can have varying fields.
    • Full Index Support: MongoDB supports indexing for efficient query execution, enhancing search speed and performance.
    • Replication & High Availability:
      • creating copies of data across multiple servers to ensure data redundancy and fault tolerance. This means that if one server fails, another can seamlessly take over, ensuring continuous availability of data.
    • Auto-Sharding:
      • a technique used for horizontal scaling, where data is distributed across multiple servers or shards.
    • Aggregation: involves the processing of data records and returning computed results. involves the processing of data records and returning computed results.
    • MongoDB Atlas: MongoDB Atlas is a fully managed cloud database service that offers scalability, security, and compliance for enterprise-level applications.
    • Various APIs: MongoDB supports a wide range of APIs for different programming languages like JavaScript, Python, Ruby, Perl, Java, Scala, C++, C#, Haskell, and Erlang.
    • Community: MongoDB has a vibrant community that contributes to its development, offering support, resources, and additional features to users.

    Features of RAI

    Rich Internet Applications (RIAs) are web-based applications that provide a user experience similar to desktop applications.

    RIAs are designed to provide a more engaging and interactive experience than traditional web applications, offering a wide range of features and functionality to enhance user engagement and productivity.

    • Direct interaction: RIAs allow users to interact directly with page elements through editing or drag-and-drop tools, providing a more engaging experience.
    • Wider range of controls: RIAs support a broader range of controls, enabling greater efficiency and enhancing the user experience.
    • Performance: Depending on the application and network characteristics, RIAs can often perform better than traditional apps, especially when processing locally on the client.
    • Animation and multimedia support: RIAs support animations, audio, and video, making them more engaging and interactive.
    • Responsiveness: RIAs are responsive, providing a graphical interface that adapts to different devices, including mobile devices.
    • No major installations: RIAs do not require significant installations, making them more accessible and convenient for users.

    REST API

    REST API, which stands for Representational State Transfer Application Programming Interface, is a type of web service architecture that follows a set of constraints to ensure a standardized and efficient communication between clients and servers.

    An architectural style for distributed hypermedia systems described by Roy Thomas Fielding in his doctoral dissertation 2000.

    REST is an architectural style, not a specification. In practice, it can be used in many different ways.

    Consists of constraints:

    • Client - Server
    • Stateless
    • Cache
    • Uniform Interface
    • Layered System
    • Code-On-Demand

    MongoDB

    MongoDB is an open source, document-oriented database designed with both scalability and developer agility in mind.

    Instead of storing your data in tables and rows as you would with a relational database, in MongoDB you store JSON-like documents with dynamic schemas(schema-free, schema less).

    MongoDB does not need any pre-defined data schema, Every document could have different data.

    Example of MongoDB data schema:

    {
	name: ben,
	age:25,
	blood: "A",
	height: 7
	weight: 60,
	alive: true
}

    Commands of MongoDB

    • db.cloneDatabase(): Copies the specified database to the current database.

      .python 
      db.cloneDatabase("https://db.url.com/db")

    • db.collectionInsertOne(): Insert single document in collection..

      .python 
      db.operateInsertOne({ 
    roll_no: 69, name: "Jonney" 
})

    • db.collectionInsertMany(name, options): Creates a new collection or view with specific options.

      .python 
      db.operateInsertMany([
    { roll_no: 69, name: "Jonney" },
    { roll_no: 60, name: "Mia" }
])

    • db.dropDatabase(): Drops the specified database and the associated data files.

      .python 
      db.dropDatabase("db_name")

    • mongod: Used to start the mongodb server.

    • show dbs: Displays the list of all databases.

    • use DATABASE_NAME: Creates and switches to a new database.

    Explain App Routing in Flask

    Flask App Routing is a feature that maps specific URLs to a specific function that handles the logic for that URL.

    This allows for more meaningful and memorable URLs, making navigation simpler for users.

    For example, in the application, the URL ('/') is associated with the home function that returns a particular string displayed on the web page.

    Function that is used for app routing:

    • app.route: routing is performed using this decorator. This decorator binds a function to a specific URL route
      • Dynamic URLs can also be built by using variables in the URL. To add variables to URLs, use <variable_name> rule.
    • add_url_rule(): is function is another approach to perform routing for the Flask web application.
    • url_for(): function is used to get generate a URL for a specific endpoint.
    • redirect(): function enables us to redirect users to different URLs.

    Example:

    .python 
    fromflaskimportFlask
app=Flask(__name__)

@app.route('/home')
defhome():
	return"Hello User";

@app.route('/home/<name>')
defhome(name):
	return"Hello, "+name;

if__name__=="__main__":
	app.run(debug=True)

    Output:

    localhost:5000/home/Jonney
    Hello, Jonney

    XMLHttpRequest

    XMLHttpRequest(XHR) objects are used to interact with servers.

    You can retrieve data from a URL without having to do a full page refresh.

    This enables a Web page to update just part of a page without disrupting what the user is doing.

    Despite its name,XMLHttpRequestcan be used to retrieve any type of data, not just XML.

    Methods of XHR:

    • XMLHttpRequest.abort() Aborts the request if it has already been sent.
    • XMLHttpRequest.getAllResponseHeaders() Returns all the response headers.
    • XMLHttpRequest.getResponseHeader() Returns the string containing the text of the specified header.
    • XMLHttpRequest.open() Initializes a request.
    • XMLHttpRequest.send() Sends the request.

    Events in XHR:

    • abort Fired when a request has been aborted, for example because the program calledXMLHttpRequest.abort().
    • error Fired when the request encountered an error.
    • load Fired when anXMLHttpRequesttransaction completes successfully.
    • progress Fired periodically when a request receives more data.
    • timeout Fired when progress is terminated due to preset time expiring.

    Example:

    .javascript 
    const xhr = new XMLHttpRequest(); 
xhr.open('GET', 'http://www.example.com/api', true); 
xhr.onload = () => if (this.status == 200) console.log(this.responseText); 
xhr.send();

    Steps to install Flask

    1. Install Python:
      1. Before installing Flask, you need to have Python installed on your system.
      2. Download Python from python.org.
    2. Install pip:
      1. Download the get-pip.py script.
      2. Run the following command:python get-pip.py

    3. Install virtualenv:

      1. Use the following command:

        .shell 
        pip install virtualenv

    4. Create a virtual environment:

      1. Create a new folder & create virtual environment in it:

        .shell 
        mkdir app
cd app
python -m venv venv

    5. Activate the virtual environment:

      1. Use the following command:

        .shell 
        venv\Scripts\activate

    6. Install Flask:

      1. Use the following command:

        .shell 
        pip install flask

    7. Create a Flask application:

      1. Create a new Python file named app.py in the root directory of your project and add the following code:

        .python 
        from flask import Flask 

app = Flask(__name__)  

@app.route('/') 
def index():
	return 'Hello, World!'  

if __name__ == '__main__':     
	app.run()

    Output:

    localhost:5000
    Hello, World
