What do you understand by DOS and DDOS attack? Explain in detail. (2023 Dec) (2023 May 5 Marks) (2022)

DoS and DDoS attacks are cyberattacks designed to disrupt the availability of a target system or network. They achieve this by overwhelming the target with excessive traffic, making it difficult or impossible for legitimate users to access the system.

  • DoS Attack
    • A DoS attack involves a single system flooding a target with traffic.
    • The attacker typically uses a single compromised system to launch the attack.
    • This makes it relatively easy to identify and mitigate the attack source.
  • DDoS Attack
    • A DDoS attack is a more sophisticated version of a DoS attack. It involves multiple systems, often controlled by a botnet, to launch the attack.
    • This makes it difficult to identify and mitigate the attack source, as the traffic appears to be coming from multiple legitimate sources.
    • Types of DDoS Attacks:
      • Volume-Based Attacks: Aim to saturate the bandwidth of the target, using techniques like ICMP floods or UDP floods.
      • Protocol Attacks: Exploit weaknesses in layer 3 and layer 4 protocols (e.g., SYN floods, Ping of Death).
      • Application Layer Attacks: Target web applications and services, aiming to exhaust resources by sending numerous legitimate-looking requests.

Explain different buffer overflow attacks also explain how to mitigate buffer overflow attack (2023 Dec) (2022) (2023 May 5 Marks)

Buffer overflow attacks occur when a program writes more data to a buffer than it can hold, leading to the overwriting of adjacent memory.

This can allow attackers to execute arbitrary code, crash the program, or corrupt data.

Types of Buffer Overflow Attacks

  1. Stack Buffer Overflow:
    • This occurs in the stack memory, typically when local variables exceed their allocated space.
  2. Heap Buffer Overflow:
    • This happens in the heap memory where dynamic memory allocation occurs.
  3. Integer Overflow:
    • When arithmetic operations exceed the maximum value that can be stored in a variable.
  4. Format String Vulnerabilities:
    • These occur when user input is improperly handled in format strings (e.g., printf). Attackers can manipulate the format string to read or write memory.

Mitigating Buffer Overflow Attacks

  1. Secure Coding Practices:
    • Input Validation: Always validate and sanitize user input to prevent malicious input from being processed.
    • Use Safe Programming Languages: Languages like Python, Java, and Rust automatically manage memory and provide built-in protections against buffer overflows.
    • Safe Library Functions: Use safe library functions that perform bounds checking, such as strncpy and snprintf.
    • Avoid Dangerous Functions: Avoid using functions like gets and sprintf that are prone to buffer overflows.
  2. Compiler-Based Protections:
    • Stack Canaries: A canary value is placed between the buffer and the return address on the stack. If the canary is overwritten, it indicates a potential attack.
    • Data Execution Prevention (DEP): Prevents the execution of code from data segments like the stack and heap, making it harder for attackers to execute malicious code.
    • Use Compiler Security Features: Compilers like GCC and Clang offer options to enable buffer overflow protections (e.g., -fstack-protector).
  3. Runtime Protection:
    • Memory Protection: Uses hardware and software techniques to protect memory regions from unauthorized access and modification.
    • Intrusion Detection Systems (IDS): Monitor network traffic and system logs for signs of attacks, including buffer overflows.
  4. Regular Software Updates:
    • Keep software up-to-date with the latest security patches to address known vulnerabilities.

Write key IT requirements for SOX and HIPAA. (2023 Dec) (2022) (2023 May 5 Marks)

SOX

SOX compliance refers to adherence to the Sarbanes-Oxley Act (SOX), a U.S. federal law enacted in 2002 in response to major corporate scandals, such as Enron and WorldCom.

It sets new standards for financial reporting for publicly traded companies. Its primary goal is to improve the accuracy and reliability of corporate financial information.

Key provisions of SOX include:

  • Audit Committee Requirements: Public companies must establish an independent audit committee to oversee the company's financial reporting processes.
  • CEO and CFO Certification: The CEO and CFO must certify the accuracy of the company's financial statements.
  • Internal Controls: Companies must establish and maintain effective internal controls to ensure the reliability of their financial reporting.
  • Prohibition of Personal Loans: Corporate officers are prohibited from receiving personal loans from the company.
  • Auditor Independence: Auditors must be independent from the companies they audit.

Non-compliance with SOX can result in significant penalties, including fines and imprisonment. Adherence to SOX is essential for maintaining investor confidence and protecting the integrity of the financial markets.

Explain SQL injection attack. State different countermeasure to prevent the attack. (2023 Dec) (2022)

A SQL injection attack is a type of cyberattack where malicious SQL code is inserted into an application's input fields to manipulate its database. By injecting malicious code, attackers can potentially steal sensitive data, modify or delete data, or even gain unauthorized access to the system.

Types of SQL Injection

  1. In-Band SQLi: The attacker can use the same channel to both launch the attack and gather results (e.g., using UNION SELECT).
  2. Inferential SQLi: No data is transferred via the web application, but the attacker reconstructs the database structure by observing the application's behavior (e.g., boolean-based or time-based).
  3. Out-of-Band SQLi: Data is retrieved using a different channel (e.g., sending the results via email or to an external server).

Countermeasures to Prevent SQL Injection:

  1. Prepared Statements and Parameterized Queries: Separate the SQL query structure from the data, preventing malicious input from being interpreted as code.
  2. Input Validation and Sanitization: Check user input for validity and remove harmful characters.
  3. Output Encoding: Properly encode output to prevent XSS attacks.
  4. Least Privilege Principle: Grant database users only the necessary permissions.
  5. Web Application Firewalls (WAFs): Filter and block malicious traffic.

Explain the term evidence and different types of evidences Where one can find it. (2023 Dec) (2023 May)

Evidence is a crucial concept in legal proceedings, serving as the foundation for establishing facts and supporting claims in court.

The term "evidence" refers to any item, material, or statement that helps to prove or disprove a matter in question.

It is derived from the Latin word "evidens," meaning clear or certain.

Types of Evidence

  1. Direct Evidence: Proves a fact outright (e.g., a witness saw a crime).
  2. Circumstantial Evidence: Suggests a fact through inference (e.g., fingerprints at a crime scene).
  3. Physical Evidence: Tangible items presented in court (e.g., weapons, documents).
  4. Documentary Evidence: Written or recorded materials (e.g., contracts, emails).
  5. Testimonial Evidence: Accounts from witnesses under oath, including expert opinions.
  6. Expert Evidence: Provided by specialists with relevant expertise (e.g., forensic analysts).
  7. Digital Evidence: Information stored digitally (e.g., data from computers, social media).
  8. Demonstrative Evidence: Visual aids to clarify facts (e.g., charts, diagrams).
  9. Hearsay Evidence: Statements made outside court; generally inadmissible due to unreliability.

What are Botnets? How it is exploit by attacker to cause cyber attack? (2022) (2023 Dec 5 Marks)

Botnets are networks of compromised computers or devices (often called "bots" or "zombies") that are controlled by an attacker to perform automated tasks. These networks can consist of thousands or even millions of infected devices, including personal computers, servers, and Internet of Things (IoT) devices.

How Botnets Work

  1. Infection: Attackers typically infect devices using malware, which can be distributed through malicious emails, compromised websites, or vulnerabilities in software. Once infected, the device becomes part of the botnet.
  2. Control: The attacker uses a command-and-control (C&C) server to manage the botnet. This server sends commands to the infected devices, directing them to carry out specific actions.
  3. Execution of Tasks: Bots can be instructed to perform various malicious activities, such as sending spam emails, stealing data, or launching attacks on target systems.

Exploitation by Attackers

  1. Distributed Denial of Service (DDoS) Attacks: Attackers can command all the bots in the network to flood a target server with traffic, overwhelming it and causing downtime.
  2. Spam Distribution: Botnets can be used to send massive amounts of spam emails, often for phishing attacks or distributing additional malware.
  3. Data Theft: Attackers can use infected devices to gather sensitive information, such as login credentials, financial information, or personal data.
  4. Cryptocurrency Mining: Some botnets are designed to use the processing power of infected devices to mine cryptocurrency, benefiting the attacker without the owner's knowledge.
  5. Credential Stuffing: Using stolen credentials, attackers can automate login attempts across various sites, exploiting accounts that have reused passwords.

What is vishing attack? How it works? How to protect from vishing attack? (2023 May)

Vishing, short for voice phishing, is a type of social engineering attack where scammers use phone calls to trick individuals into revealing sensitive information like passwords, credit card numbers, or bank details.

How Vishing Works

  1. Impersonation: The scammer poses as a legitimate entity, such as a bank, credit card company, or government agency. ¬†
  2. Urgency and Fear: They often create a sense of urgency or fear, claiming that there's an issue with your account or that you're facing legal trouble. ¬†
  3. Information Gathering: The scammer will ask you to verify personal information, such as your Social Security number, birth date, or account details. ¬†
  4. Phishing for Sensitive Information: They may ask you to provide sensitive information like passwords or one-time codes.

Protecting Yourself from Vishing Attacks

  • Be Skeptical of Unsolicited Calls: Never share personal information with anyone who calls you unexpectedly, even if they claim to be from a reputable organization. ¬†
  • Verify the Caller's Identity: If you receive a suspicious call, hang up and call the company directly using a number you know to be legitimate. ¬†
  • Don't Provide Sensitive Information Over the Phone: Legitimate organizations will never ask you to share sensitive information like passwords or account numbers over the phone. ¬†
  • Be Wary of Spoofed Numbers: Scammers can spoof caller ID to make it appear as if they're calling from a legitimate number. ¬†
  • Use Strong Passwords and Two-Factor Authentication: Strong passwords and two-factor authentication can help protect your accounts, even if your information is compromised. ¬†
  • Be Cautious of High-Pressure Tactics: Legitimate organizations won't pressure you to make quick decisions or share sensitive information. ¬†
  • Educate Yourself: Stay informed about the latest scams and phishing techniques.
  • Use Call Blocking and Spam Filtering: Consider using call-blocking and spam-filtering services to reduce the number of unsolicited calls you receive.

What are basic security precautions to be taken to safeguard Laptops and Wireless devices? Explain. (2023 May)

Physical Security

  1. Use Strong Passwords: Set a strong password for your device and any accounts. Use a mix of letters, numbers, and symbols.
  2. Lock Your Device: Always lock your laptop or device when not in use, especially in public places.
  3. Physical Locks: Consider using a cable lock for your laptop when working in public spaces to prevent theft.

Software Security

  1. Keep Software Updated: Regularly update your operating system, applications, and antivirus software to protect against vulnerabilities.
  2. Install Antivirus/Anti-malware Software: Use reputable antivirus software and keep it updated to detect and eliminate threats.
  3. Use a Firewall: Enable the built-in firewall on your device to help block unauthorized access.
  4. Enable Encryption: Use full disk encryption (like BitLocker for Windows or FileVault for macOS) to protect your data in case of theft.

Network Security

  1. Secure Your Wi-Fi Network: Use a strong password for your Wi-Fi network and enable WPA3 encryption if available.
  2. Avoid Public Wi-Fi for Sensitive Activities: Refrain from accessing sensitive accounts or conducting financial transactions over public Wi-Fi.
  3. Use a VPN: A Virtual Private Network (VPN) can help secure your internet connection and protect your data from eavesdropping on public networks

General Best Practices

  1. Regular Backups: Back up important data regularly, either to an external drive or cloud storage, to prevent data loss.
  2. Be Cautious with Downloads: Only download software and apps from trusted sources. Be wary of email attachments and links.
  3. Educate Yourself: Stay informed about the latest security threats and best practices to protect your devices.

What is Cybercrime? Who are Cybercriminals? Explain. (2023 May)

Cybercrime refers to illegal activities that involve computers or networks. This can include a wide range of offenses, such as:

  • Hacking: Unauthorized access to computer systems or networks.
  • Identity Theft: Stealing personal information to impersonate someone else.
  • Phishing: Fraudulent attempts to obtain sensitive information by disguising as a trustworthy entity.
  • Malware Distribution: Spreading malicious software designed to damage or disrupt systems.
  • Ransomware: Encrypting a victim's data and demanding payment for its release.
  • Online Fraud: Scams conducted via the internet, including auction fraud or investment scams.
  • Cyberbullying: Harassment or intimidation through digital communication.
  • Denial-of-Service Attacks: Overwhelming a website or service to make it unavailable.

Cybercriminals are individuals or groups that engage in cybercrime. They can vary widely in terms of their motivations, skills, and methods:

  1. Hackers: Often highly skilled in programming and network systems, they may hack for various reasons, from financial gain to political activism (hacktivism).
  2. Organized Crime Groups: These are well-structured groups that operate like traditional criminal organizations but focus on cyber activities, often involved in large-scale fraud and identity theft.
  3. Insiders: Employees or individuals with access to a company's systems who misuse their privileges for personal gain or to harm the organization.
  4. Nation-State Actors: Government-sponsored groups that engage in cyber warfare or espionage against other nations, often targeting critical infrastructure or sensitive information.
  5. Lone Wolves: Individuals acting alone, sometimes driven by personal grievances or ideological beliefs, who may conduct cyberattacks without the backing of a larger organization.

Explain what is Information Security Standard

Information Security Standards are a set of guidelines and best practices designed to protect sensitive information from unauthorized access, use, disclosure, disruption, modification, or destruction.

These standards provide a framework for organizations to implement security controls and manage risks associated with information security.

Key Information Security Standards:

  • ISO/IEC 27001: This is a widely recognized international standard that provides a framework for establishing, implementing, maintaining, and continually improving an Information Security Management System (ISMS).
  • NIST Cybersecurity Framework (CSF): Developed by the National Institute of Standards and Technology (NIST), the CSF provides a risk-based approach to managing cybersecurity risk. It consists of five core functions: Identify, Protect, Detect, Respond, and Recover. ¬†
  • PCI DSS (Payment Card Industry Data Security Standard): This standard is designed to protect cardholder data. It requires organizations that process, store, or transmit cardholder data to implement specific security controls. ¬†
  • HIPAA (Health Insurance Portability and Accountability Act): This US law requires organizations that handle protected health information (PHI) to implement specific security measures.¬†
  • GDPR (General Data Protection Regulation): This EU regulation governs the processing of personal data within the EU and EEA. It imposes strict data protection requirements on organizations.

Benefits of Adhering to Information Security Standards:

  • Enhanced Security: Implementing these standards helps organizations strengthen their security posture and protect sensitive information. ¬†
  • Reduced Risk: By following best practices, organizations can minimize the risk of data breaches and cyberattacks. ¬†

Steganography (2023 May)

Steganography is the practice of concealing a message within another message or a file, making the hidden message undetectable. Unlike cryptography, which focuses on encrypting the message to make it unreadable, steganography aims to hide the very existence of the message.

How Does Steganography Work?

  1. Carrier File: This is the file that will conceal the secret message. It could be an image, audio file, or even a text document.
  2. Secret Message: The information to be hidden.
  3. Steganographic Algorithm: This algorithm embeds the secret message into the carrier file in a way that's difficult to detect.

Common Techniques:

  • Least Significant Bit (LSB) Steganography: This involves modifying the least significant bit of each pixel in an image to store the secret message.
  • Audio Steganography: Similar to LSB steganography, this technique modifies the least significant bits of audio samples.
  • Text Steganography: This involves hiding information within the formatting of a text document, such as font size, color, or spacing.

Applications of Steganography

  • Secure Communication: Governments and military organizations can use steganography to transmit secret messages without raising suspicion.
  • Digital Watermarking: Artists and content creators can embed copyright information or digital signatures into their work.
  • Data Hiding: Sensitive data can be hidden within seemingly innocuous files to protect it from unauthorized access.

Trojan horse and backdoor (2023 May)

Trojan Horse

A Trojan horse is a type of malicious software disguised as legitimate software. It often comes in the form of a seemingly harmless program, such as a game, utility, or software update. Once installed, it can perform various malicious activities without the user's knowledge, including:

  • Data theft: Stealing sensitive information like passwords, credit card numbers, and personal data.
  • System damage: Corrupting or deleting files, or rendering the system unusable.
  • Backdoor installation: Creating a backdoor to allow remote access to the system.
  • Botnet creation: Turning the infected device into part of a botnet, a network of compromised devices controlled by a malicious actor.

Types of Trojan Horse:

  • Exploit Trojans - These Trojans inject a machine with code deliberately designed to take advantage of a weakness inherent to a specific piece of software.
  • Rootkit Trojans - These Trojans are intended to prevent the discovery of malware already infecting a system so that it can affect maximum damage.
  • SMS Trojan - These programs can cost you money by sending text messages from your mobile device to premium rate phone numbers.
  • Distributed Denial of Service (DDoS) Trojans - These are programmed to execute DDoS attacks, where a network or machine is disabled by a flood of requests originating from many different sources.
  • Downloader Trojans - These are files written to download additional malware, often including more Trojans, onto a device.

Backdoor

A backdoor is a secret entry point into a computer system that bypasses normal authentication procedures. It allows an attacker to gain unauthorized access to the system remotely. Backdoors can be installed through various means, including:

  • Malicious software: Trojans, worms, and viruses can install backdoors.
  • Software vulnerabilities: Exploiting vulnerabilities in software can create backdoors.
  • Misconfigurations: Incorrect system configurations can leave backdoors open.

Short notes on Active attack and Passive attack.

Active Attacks

Active attacks involve the attacker directly modifying or disrupting the target system or network. Some key characteristics of active attacks:

  • The attacker attempts to change or modify the content of messages
  • They are dangerous to both integrity and availability of data
  • The victim is informed about the attack
  • System resources can be changed or damaged
  • Examples include masquerade, message modification, replay, and denial of service attacks

Passive Attacks

In passive attacks, the attacker simply observes and gathers information without altering anything. Key points about passive attacks:

  • The attacker monitors and copies message contents without modifying them
  • They are a threat to confidentiality but not integrity or availability
  • The victim is not informed about the attack
  • System resources remain unchanged
  • Examples include eavesdropping and traffic analysis

Explain requirements for PCI-DSS compliance. Also give the benefits of it.

PCI-DSS (Payment Card Industry Data Security Standard) is a set of security requirements designed to protect cardholder data.

It is mandated by major credit card companies like Visa, Mastercard, American Express, Discover, and JCB.

Key Requirements of PCI-DSS

  1. Install and maintain a firewall: This is to protect cardholder data from unauthorized access.
  2. Do not store full cardholder data: Only store the last four digits of the card number, expiration date, and service code.
  3. Protect stored cardholder data: If you must store cardholder data, encrypt it.
  4. Develop, implement, and maintain secure systems and applications: Regularly update software to patch vulnerabilities.
  5. Protect cardholder data in transmission: Use strong encryption protocols like HTTPS.

Benefits of PCI-DSS Compliance

  • Reduced risk of data breaches: Adhering to PCI-DSS can significantly reduce the risk of data breaches that could lead to financial loss, reputational damage, and legal consequences.
  • Enhanced customer trust: Demonstrating PCI-DSS compliance can boost customer confidence in your organization's ability to protect their sensitive information.
  • Improved regulatory compliance: PCI-DSS often aligns with other industry regulations and standards, simplifying compliance efforts.
  • Lowered insurance premiums: Many insurance companies offer discounts to businesses that are PCI-DSS compliant.
