(a) Define Graph. List down the applications of graph.

A graph is a mathematical representation of a set of objects, where some pairs of the objects are connected by links. The objects are represented by vertices (or nodes) and the links are represented by edges. Graphs are used to model many different types of relationships and processes in physical, biological, social, and information systems.

Some applications of graph:

  1. Computer graphics and image processing
  2. Artificial intelligence
  3. Web and Information Retrieval
  4. Cryptography
  5. Recommendation systems
  6. Machine learning and data mining
  7. Social Media Analytics
  8. Computer Networking
  9. Natural Language Processing
  10. Route finding and navigation

(b) Explain winding and unwinding phase of recursion.

Recursion is a method of solving a problem by breaking it down into smalle, simpler subproblems.

The winding phase of recursion refers to the process of breaking down a problem into smaller subproblems and making recursive calls to slove them.

The unwinding phase of recursion refers to the process of combining the solutions of the smaller subproblems to solve the original problem. This typically happens when the recursion reahes the base case, which is the point at which the problem can no longer be broken down further.

For example, consider a recursive function that finds the factorial of a given number. The winding phase would be the process of making recursive calls to find the factorial of smaller numbers, while the unwinding phase would be the process of combining the solutions of these smaller factorials to find the final factorial of the original number.

Winding phase starts when the function is called and ends with the base case, unwinding phase starts when the base case is hit and ends when the final output is obtained.

(c) Explain the concept of Buddy system.

The buddy system is a memory management technique used in computer systems to efficiently manage memory allocation and deallocation. The basic idea behind the buddy system is to divide memory into blocks of different sizes called "buddies", that can be allocatd or deallocated as needed.

The buddy system works by dividing memory into blocks of a certain size, and then further dividing each block into two "buddies" of half the size. When a block of memory is needed, the buddy system searches for the smallest block that can satisfy the request. If a block is not available, the buddy system will split a larger block into two smaller blocks, one of which will be used to satisfy the request.

(d) Explain different types of link list.

There are three types of linked list:

Singly Linked List

It is the simplest type of linked list in which every node contains some data and a pointer to the next node of the same data type. The node contains a pointer to the next node means that the node stores the address of the next node in the sequence. A single linked list allows the traversal of data only in one way.

Doubly Linked List

A doubly linked list or a two-way linked list is a more complex type of linked list that contains a pointer to the next as well as the previous node in sequence, Therefore, it contains three parts of data, a pointer to the next node, and a pointer to the previous node. This would enable us to traverse the list in the backward direction as well.

Circular Linked List

A circular linked list is one in which the last node contains the pointer to the first node of the list. While traversing a circular linked list, one can begin at any node and traverse the list in any direction forward and backward until reaching the same node where it started. Thus, a circular linked list has no beginning and no end.

(b) Explain selection sort with example by giving its algorithm and comment on its complexity.

Selection sort is a simple sorting algorithm that repeatedly selects the smallest element from the unsorted portion of the list and places it at the end of the sorted portion of the list.

The algorithm for selection sort can be outlined as follows:

  • Step 1 Start by assuming the first element in the list is the smallest.
  • Step 2 Iterate through the list, comparing each element to the current minimum. I f a smaller element is found, set it as the new minimum.
  • Step 3 Once the end of the list is reached, swap the position of the minimum el ement with the first unsorted element.
  • Step 4 Repeat steps 1-3 until the entire list is sorted.
