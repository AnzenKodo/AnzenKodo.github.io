1. Data stream Management system.

A data stream is a flow of data which arrives at uncertain intervals of time.

A Data Stream Management System (DSMS) is a software system designed to manage, process, and analyze continuous, high-velocity data streams in real time.

  1. Input Streams: One or more streams with varying data types and flow rates.
  2. Stream Processor: Performs operations like sampling, cleaning, filtering, and querying on input streams.
  3. Working Storage: Holds essential parts of input streams for fast query processing.
  4. Archival Storage: Large storage for archiving streams.
  5. Output Streams: Consists of processed streams and query results.

2. Stream Sources / Queries & Issues in stream processing

Stream Sources:

  1. Sensor Imagery Data: Sensors (e.g., temperature, surface height) produce real number streams at varying rates.
  2. Graphical Image Data: This is generally recorded by satellites sending a lot of images ranging from many terabytes of images recorded per day.
  3. Internet Data: Search engine queries (e.g., Google receives hundreds of millions daily). Web traffic (clicks). An increase in specific queries might indicate a virus.

Stream Queries

  1. Standing Queries:
    • Permanently stored queries in the stream processor.
    • Answered at regular intervals.
    • Examples:
      • Maximum temperature from an ocean sensor (update maximum on new readings).
      • Average temperature of all stream values (update average and count on new readings).
  2. Ad-Hoc Queries:
    • Spontaneous queries.
    • Require an interface for on-demand querying.
    • Systems generally do not store entire streams.
    • Answering may involve storing parts or summaries.
    • Example: Number of registrations for a social networking site in the past three months (requires storing registration time).

Issues in Stream Processing

  1. High Speed Stream:
    • Stream generates elements at a much faster rate compared to normal databases.
    • If we don't process them efficiently in real time, we lose the opportunity or in other words stream data would be lost.

  2. Parallel stream execution:

    • In some cases streams may be slow in generation but many streams could have been opened at the same time.

    • So, the one stream would have used main memory for execution, all the other possible streams would be following the same regime.

      3. Sampling Techniques.

Sampling is a method that allows us to get information about the population based on the statistics from a subset of the population (sample), without having to investigate every individual.

Types of sampling techniques available to us:

  • Probability Sampling: Every element of the population has an equal chance of being selected.
    • Simple Random Sampling: Each member of the population has an equal chance of being selected.
    • Systematic Sampling Selects samples at regular intervals from an ordered list.
    • Stratified Sampling: Divides the population into subgroups (strata) based on characteristics and samples from each.
    • Cluster Sampling Divides the population into clusters and randomly selects entire clusters.
  • Non-Probability Sampling: All elements do not have an equal chance of being selected.
    • Convenience Sampling Selects data points that are easily accessible.
    • Inverse Sampling: Used in big data to adjust for selection bias by incorporating auxiliary information.
    • Data Integration: Combines big data samples with independent probability samples to reduce bias.

4. DGIM algorithm.

There are six rules that must be followed when representing a stream by buckets.

  • The right end of a bucket is always a position with a 1.
  • Every position with a 1 is in some bucket.
  • No position is in more than one bucket.
  • There are one or two buckets of any given size, up to some maximum size.
  • All sizes must be a power of 2.
  • Buckets cannot decrease in size as we move to the left (back in time).

9. SVM algorithm

SVM stands for Support Vector Machine, which is a supervised machine learning algorithm used for classification and regression tasks.

SVMs can be used for a variety of tasks, such as text classification, image classification, spam detection,¬†handwriting identification, gene expression analysis, face detection, and anomaly detection

Types of Support Vector Machine

  • Linear SVM:¬†
    • Linear SVMs use a linear decision boundary to separate the data points of different classes.
  • Non-Linear SVM:¬†
    • Non-Linear SVM can be used to classify data when it cannot be separated into two classes by a straight line (in the case of 2D).

Support Vector Machine Terminology

  1. Data Points: There are two classes of data points represented by different shapes and colors.
  2. Hyperplane: A hyperplane is like a decision boundary that separates different classes in the data.
  3. Positive Hyperplane and Negative Hyperplane: The positive hyperplane passes through the support vectors of the positive class, and the negative hyperplane passes through the support vectors of the negative class.
  4. Support Vectors: Support vectors are the data points closest to the hyperplane.
  5. Maximum Margin: The margin is the distance between the hyperplane and the closest data points.

Parallel SVM

"Parallel SVM" generally refers to techniques that aim to speed up the training or prediction process of Support Vector Machines (SVMs) by utilizing parallel computing resources.

10. KNN algorithm

The¬†K-Nearest Neighbors (KNN) algorithm¬†is a non-parametric, supervised learning method used for both classification and regression tasks in machine learning.

How it works:

  1. Choose K: Decide on the number of nearest neighbors to consider.
  2. Calculate Distance: For a new, unseen data point, calculate the distance to every other data point in the training set.
  3. Find Nearest Neighbors: Identify the 'K' training data points that have the smallest distances to the new data point.
  4. Make Prediction:
    • Classification: Assign the new data point to the class that is most frequent among its 'K' nearest neighbors (majority vote).
    • Regression: Predict the value of the new data point as the average (or weighted average) of the values of its 'K' nearest neighbors.

Advantages of KNN Algorithm:

  • It is simple to implement.
  • It is robust to the noisy training data
  • It can be more effective if the training data is large.

Disadvantages of KNN Algorithm:

  • Always needs to determine the value of K which may be complex some time.
  • The computation cost is high because of calculating the distance between the data points for all the training samples.

12. Structure of web

Recently, a new map of cyberspace shows that the Web resembles a bow tie, with divisive boundaries that cherspace shows that between regions is difficult or even impossible, according to a new study conducted and published by researchers at AltaVista, Compaq, and IBM.

Previous theories had a belief that the Web is highly connected, with no more than 19 degrees of separation from any one site to another.

Links are basically acts as the glue that connect pages on the Web, and they're a vital source for navigation.

  • Strongly Connected Component (SCC): This is the central part of the bow-tie structure, where every page can reach every other page through a directed path.
  • IN (Origination) Component: This component consists of web pages that can reach the SCC by following hyperlinks, but cannot be reached from the SCC.
  • OUT (Termination) Component: his component includes web pages that can be reached from the SCC by following hyperlinks, but cannot link back to the SCC.
  • Tendrils: These are pages that can reach or be reached by either the IN or OUT components but not both.
  • Disconnected Components: These are pages that neither link to nor are linked from the SCC or any other components.

15. Recommendation system & Types .

A recommendation system is a type of machine learning system that provides personalized recommendations to users based on their past behaviors, preferences, and patterns.

Recommendation systems are widely used in e-commerce, social media, entertainment, and other online platforms to increase user engagement and retention, improve customer satisfaction, and drive sales and revenue

Types

  • Content-Based Filtering:
    • Content-based recommendation systems recommend items to users based on their past preferences and behaviors.
    • This type of system analyzes the user‚Äôs historical data, such as their search history, browsing history, or purchase history, and recommends items that are similar to the ones the user has interacted with before.
  • Collaborative Filtering
    • Collaborative filtering recommendation systems recommend items to users based on the preferences and behaviors of other similar users.
    • Two kinds of collaborative filtering techniques used are
      • User-User collaborative filtering is a type of recommendation system that makes predictions for a user based on the preferences of similar users.
      • Item-Item collaborative filtering, on the other hand, recommends items to a user based on the preferences for similar items.
