What is clique percolation method Write an algorithm on (CPM). Also show how the CPM finds clique for the following graph. Explain with steps (10 Marks, 10 Marks, 10 Marks) (2024 December, 2024 May, 2023 December)

The Clique Percolation Method (CPM) is an algorithm used to find overlapping communities in networks. It identifies communities based on the concept of k-cliques, where a k-clique is a complete subgraph of k nodes.

Algorithm for CPM:

  1. Find all maximal cliques in the graph. ¬†
  2. Identify k-cliques: Extract all cliques of size k. ¬†
  3. Create a clique graph: Construct a graph where each node represents a k-clique, and an edge connects two nodes if the corresponding k-cliques share k-1 nodes. ¬†
  4. Find connected components: The connected components in the clique graph represent the communities.

Explain DGIM algorithm (10 Marks, 10 Marks, 10 Marks, 5 Marks) (2024 December, 2024 May, 2023 December, 2023 May).

The Datar-Gionis-Indyk-Motwani (DGIM) Algorithm is used to estimate the number of 1's in a stream within a window of N bits. It uses O(log¬≤ N) bits to represent a window of N bits and estimates the number of 1's with an error of no more than 50%.

There are six rules that must be followed when representing a stream by buckets.

  1. The right end of a bucket is always a position with a 1.
  2. Every position with a 1 is in some bucket.
  3. No position (1's, 0's) can be in more then one bucket.
  4. There are one or two buckets of any given size (which must be a power of 2).
  5. All buckets should be power of 2.
  6. Buckets cannot decrease in size as we move to the left (back in time).

$$\text{No of 1's: } 2^2 + 2^2 + 2^1 = 4 + 4 + 2 = 10 $$ There are 10 bits of 1's in the given dataset.

Explain CURE algorithm, clearly stating its advantages over traditional clustering algorithm (10 Marks, 10 Marks, 10 Marks, 10 Marks) (2024 December, 2024 May, 2023 December, 2023 May).

  • This algorithm, called Clustering Using Representatives (CURE)
  • Clustering is classification of objects into groups
  • CURE algorithm is hierarchical, which means it starts with single point cluster and move upward to merge cluster until desired number is reached.
  • It can detect cluster with non-spherical shape.
  • Works very nicely with outliers.
  • Efficient for working with large dataset.
  • Architecture:
  • Procedure:
    1. Select target sample numbers.
    2. Choose C well scattered points in a cluster.
    3. Those scattered point are strunk toward centroid.
    4. Then these points are used as representatives of clusters.
    5. After merging the points new sample points will be selected.
    6. Cluster merging will stop until target k is reached.

Advantages of CURE:

  • Handles Non-Spherical Shapes: Traditional algorithms often assume clusters are spherical. CURE can identify clusters of arbitrary shapes, including elongated or irregular ones. ¬†
  • Robust to Outliers: CURE is less sensitive to outliers.
  • Scalability for Large Datasets: CURE employs a combination of random sampling and partitioning of the data, which makes it more efficient and scalable for clustering large databases that may not fit entirely in main memory.
  • Better Cluster Boundary Recognition: Using multiple representative points helps CURE to better recognize and define the boundaries between clusters.

Explain Issues in Data stream query processing (10 Marks, 5 Marks) (2024 December, 2023 December).

Issues in Data Stream Query Processing:

  • Continuous Data Arrival: Streams are unbounded and continuous, requiring queries to process data in real-time as it arrives, unlike traditional batch processing of finite datasets.
  • High Data Arrival Rates: Data can arrive at very high velocities, demanding efficient processing techniques to keep up with the flow and avoid data loss or significant delays.
  • Limited Memory: Processing large volumes of streaming data typically requires algorithms that use limited memory, as storing the entire stream is often infeasible. This necessitates techniques like windowing and summarization.
  • Latency: Many stream processing applications have strict latency requirements, demanding query processing with minimal delay from data arrival to result generation.
  • Scalability: As data rates and volumes can vary significantly, stream processing systems and query processing techniques must be highly scalable to handle these fluctuations.

Explain MapReduce programming model in detail (5 Marks, 5 Marks) (2024 May, 2024 December).

The MapReduce programming model is a framework for processing large datasets in parallel across a distributed computing environment. It involves two primary functions:

  1. The Map Task:
    • Input: The Map task takes a portion of the input data as its input. This input data is typically split into chunks.
    • Processing: This function processes each input key-value pair independently.
    • Output: The Map function emits a set of intermediate key-value pairs.
    • Example:
      • ("the", 1)
      • ("quick", 1)
      • ("brown", 1)
      • ("the", 1)
  2. The Reduce Task:
    • Input: Before reaching the Reduce task, all the intermediate key-value pairs generated by the various Map tasks are:
      • Shuffled: Grouped together based on their intermediate keys.
      • Sorted: Within each group of the same key, the values are typically sorted.
    • Processing: This function takes an intermediate key and an iterator of all the values associated with that key as its input.
    • Output: The Reduce function generates the final output, which is also in the form of key-value pairs.
    • Example:
      • ("the", 2)
      • ("quick", 1)
      • ("brown", 1)

Modified PageRank algorithms are variations of the original PageRank algorithm that incorporate additional factors or make adjustments to improve the ranking of web pages in specific contexts.

Common Modifications and Considerations:

  • Weighted Links: Instead of treating all links equally, modified algorithms might assign different weights to links based on the source page's authority, topic relevance, or the position of the link on the page.

  • Content Relevance: Certain modified algorithms integrate content analysis. They might boost the rank of pages whose content is highly relevant to the pages linking to them.
  • Time: Newer, actively updated pages might receive a higher rank.
  • User Behavior: Some advanced modifications might incorporate data about how users interact with search results, such as click-through rates, to influence page rankings.

Explain Column family store and Graph Store NoSQL architectural pattern with example (10 Marks) (2024 December).

Column Family Store

  • Column family stores are NoSQL databases that organize data into columns and column families. A column family is a collection of related columns. Unlike relational databases where you have rows and fixed columns, column family stores offer a flexible schema where different rows can have different columns.
  • Key characteristics:
    • Data is stored in column families, which are similar to tables in relational databases.
    • Within a column family, data is organized into rows, each with a unique key.
    • Each row consists of a set of columns, which can vary from row to row.
    • Columns are grouped into column families for storage and retrieval.

Graph Store

  • Graph stores are NoSQL databases that use a graph data model to represent and store data. A graph data model focuses on relationships between data points.
  • Key characteristics:
    • Data is represented as nodes (entities) and edges (relationships)
    • Nodes store data entities, similar to rows in a table.
    • Edges represent the connections between nodes.
    • Graph databases are optimized for traversing and querying relationships.

Explain PCY algorithm and its 2 types with neat labeled diagram (10 Marks, 10 Marks, 10 Marks) (2024 December, 2023 December, 2023 May).

The PCY algorithm is used to reduce the size of candidate sets in the A-Priori algorithm by utilizing unused space in main memory during the first pass. It employs a hash table to count pairs of items and identifies frequent buckets to more accurately determine candidate pairs. ¬†

Types of the PCY Algorithm:

  • Pass 1:
    • Item counts: During the first pass over the transaction data, the algorithm counts the occurrences of each individual item.
    • Hash table for pairs: Simultaneously, as transactions are read, the algorithm considers all pairs of items within each transaction.
    • Bitmap: Each bucket in the hash table corresponds to a bit in the bitmap. If the count of pairs that hash to a particular bucket exceeds a certain threshold (support count), the corresponding bit in the bitmap is set to 1; otherwise, it's set to 0.
  • Pass 2:
    • Frequent items: The frequent items identified in Pass 1 (those meeting the minimum support count) are used to generate candidate pairs for the second pass.
    • Bitmap (used for pruning): A candidate pair (i,j) is only considered for counting in the second pass if both i and j were frequent in the first pass and their hash bucket in Pass 1 had a count above the threshold (i.e., the corresponding bit in the bitmap is 1).
    • Counts of candidate pairs: For the candidate pairs that pass the bitmap check, the algorithm counts their actual occurrences in the transaction data during the second pass.

Recall all NoSQL design patterns with example. Justify CAP property (10 Marks, 5 Marks, 10 Marks, 5 Marks, 10 Marks) (2024 May, 2023 December, 2023 December, 2023 May, 2023 May).

  • Key-Value Store:
    • Simple pattern using keys to retrieve associated values. Optimized for lookups.
    • Example: Session management in web applications where session IDs are keys and session objects are values.
  • Document Database:
    • Stores data as JSON-like documents with flexible schemas. Good for semi-structured data.
    • Example: Product catalogs where each product is a document with attributes like name, description, price, and reviews.
  • Column-Family Database:
    • Organizes data into column families, which are groups of related columns. Efficient for sparse data.
    • Example: Storing user profiles where column families might include personal information, contact details, and purchase history.
  • Graph Database:
    • Models data as nodes and relationships (edges). Ideal for interconnected data.
    • Example: Social networks where users are nodes and friendships are edges, enabling efficient traversal of relationships.

The CAP Theorem:

  • The CAP theorem states that it is impossible for a distributed system to simultaneously provide all three of the following guarantees:
    • Consistency:
      • Every read receives the most recent write or an error
      • RDBMS
    • Availability:
      • Every request receives a (non-error)3 response, without the guarantee that it contains the most recent write
      • MongoDB, HBase
    • Partition Tolerance:
      • The system continues to operate despite4 arbitrary partitioning due to network failures
      • Cassandra, CouchDB
  • In simpler terms, in the presence of a network partition, one has to choose between consistency and availability.
  • Basically, you can't have it all. No distributed database system can guarantee all three.

How NoSQL systems guarantees BASE property (5 Marks, 5 Marks) (2023 December, 2023 May).

BASE stands for Basically Available, Soft state, Eventually consistent

BASE property is an acronym describing characteristics of some NoSQL databases, contrasting with ACID (Atomicity, Consistency, Isolation, Durability) properties often associated with traditional relational databases.

How NoSQL Systems Guarantee BASE Property:

  • Basically Available:
    • NoSQL systems often achieve high availability through techniques like replication and distributed architectures.
    • If one node fails, other nodes can still serve read and write requests.
  • Soft State:
    • Data changes might take time to be replicated across all nodes.
    • During this time, different clients might see different versions of the data.
    • The system is in a state of flux ("soft") until consistency is achieved. ¬†
  • Eventually Consistent: The system guarantees that if no new updates are made to the data item, eventually all accesses to that item will return the latest updated value.

What are the advantages and limitations of Hadoop (5 Marks) (2023 December).

Advantages of Hadoop:

  • Scalability: Can handle petabytes or even exabytes of data by adding more nodes to the cluster.
  • Cost-Effective: Uses commodity hardware, making it a relatively inexpensive solution for storing and processing large datasets.
  • Parallel Processing: MapReduce framework enables parallel processing of data across the cluster, significantly reducing processing time.
  • Flexibility: Can store various types of data, including structured, semi-structured, and unstructured data.
  • Open Source: Large and active community support, with continuous development and a wide range of tools and libraries available.

Limitations of Hadoop:

  • Complexity: Setting up, configuring, and managing a Hadoop cluster can be complex.
  • Resource Intensive: MapReduce jobs can be resource-intensive, requiring significant disk I/O and network bandwidth.
  • Not Ideal for Small Datasets: Overhead of distributing tasks across the cluster can make it inefficient for processing small amounts of data.
  • Security Concerns: While security features exist, configuring and managing them properly can be challenging.

Structure of the web (10 Marks) (2023 December).

Recently, a new map of cyberspace shows that the Web resembles a bow tie, with divisive boundaries that cherspace shows that between regions is difficult or even impossible, according to a new study conducted and published by researchers at AltaVista, Compaq, and IBM.

Previous theories had a belief that the Web is highly connected, with no more than 19 degrees of separation from any one site to another.

Links are basically acts as the glue that connect pages on the Web, and they're a vital source for navigation.

  • Strongly Connected Component (SCC): This is the central part of the bow-tie structure, where every page can reach every other page through a directed path.
  • IN (Origination) Component: This component consists of web pages that can reach the SCC by following hyperlinks, but cannot be reached from the SCC.
  • OUT (Termination) Component: his component includes web pages that can be reached from the SCC by following hyperlinks, but cannot link back to the SCC.
  • Tendrils: These are pages that can reach or be reached by either the IN or OUT components but not both.
  • Disconnected Components: These are pages that neither link to nor are linked from the SCC or any other components.

Summarize Bloom's filter with example and its applications (10 Marks) (2023 December)

It was invented by Burton Howard Bloom in 1970.

Its primary purpose is to quickly determine if an element might be in a set or is definitely not in a set, while using significantly less memory than traditional set data structures like hash tables or lists

Components

  • Bit Array: An array of n bits initialized to 0's
  • Hash Function: A set 'H' of 'k' hash functions each of which maps a key to one of the 'n' bit locations h1, h2,...hk
  • Insertion: To add an element, it's passed through each hash function.
  • Membership Testing: To check if an element is in the set, it's passed through the same hash functions
    • If any of these bits are 0, the element is definitely not in the set.
    • If all these bits are 1, the element might be in the set (a true positive) or the bits could have been set by other elements (a false positive).

Diagram

  • Initially all the locations in the array are set to 0.
  • Each key 'k' is taken from S and one by one all of the 'k' hash functions in H are applied to this K.
  • All the bit locations produced by n1(k) are set to 1.

Example

Imagine a Bloom filter with a bit array of 10 bits and 3 hash functions. We want to add the words "apple".

  • hash1("apple") % 10 might give 2.
  • hash2("apple") % 10 might give 5.
  • hash3("apple") % 10 might give 8.
  • The bits at indices 2, 5, and 8 are set to 1.

Application

  • Database Systems: Reducing disk lookups for non-existent keys.
  • Network Security: Identifying malicious URLs or blocked IP addresses.
  • Content Delivery Networks (CDNs): Determining if a requested piece of content is likely present in a cache.
  • Recommendation Systems: Filtering out already seen items to avoid redundant recommendations.

A data stream is a flow of data which arrives at uncertain intervals of time.

A Data Stream Management System (DSMS) is a software system designed to manage, process, and analyze continuous, high-velocity data streams in real time.

  1. Input Streams: One or more streams with varying data types and flow rates.
  2. Stream Processor: Performs operations like sampling, cleaning, filtering, and querying on input streams.
  3. Working Storage: Holds essential parts of input streams for fast query processing.
  4. Archival Storage: Large storage for archiving streams.
  5. Output Streams: Consists of processed streams and query results.

2. Stream Sources / Queries & Issues in stream processing

Stream Sources:

  1. Sensor Imagery Data: Sensors (e.g., temperature, surface height) produce real number streams at varying rates.
  2. Graphical Image Data: This is generally recorded by satellites sending a lot of images ranging from many terabytes of images recorded per day.
  3. Internet Data: Search engine queries (e.g., Google receives hundreds of millions daily). Web traffic (clicks). An increase in specific queries might indicate a virus.

Stream Queries

  1. Standing Queries:
    • Permanently stored queries in the stream processor.
    • Answered at regular intervals.
    • Examples:
      • Maximum temperature from an ocean sensor (update maximum on new readings).
      • Average temperature of all stream values (update average and count on new readings).
  2. Ad-Hoc Queries:
    • Spontaneous queries.
    • Require an interface for on-demand querying.
    • Systems generally do not store entire streams.
    • Answering may involve storing parts or summaries.
    • Example: Number of registrations for a social networking site in the past three months (requires storing registration time).

9. SVM algorithm

SVM stands for Support Vector Machine, which is a supervised machine learning algorithm used for classification and regression tasks.

SVMs can be used for a variety of tasks, such as text classification, image classification, spam detection,¬†handwriting identification, gene expression analysis, face detection, and anomaly detection

Types of Support Vector Machine

  • Linear SVM:¬†
    • Linear SVMs use a linear decision boundary to separate the data points of different classes.
  • Non-Linear SVM:¬†
    • Non-Linear SVM can be used to classify data when it cannot be separated into two classes by a straight line (in the case of 2D).

Support Vector Machine Terminology

  1. Data Points: There are two classes of data points represented by different shapes and colors.
  2. Hyperplane: A hyperplane is like a decision boundary that separates different classes in the data.
  3. Positive Hyperplane and Negative Hyperplane: The positive hyperplane passes through the support vectors of the positive class, and the negative hyperplane passes through the support vectors of the negative class.
  4. Support Vectors: Support vectors are the data points closest to the hyperplane.
  5. Maximum Margin: The margin is the distance between the hyperplane and the closest data points.

Parallel SVM

"Parallel SVM" generally refers to techniques that aim to speed up the training or prediction process of Support Vector Machines (SVMs) by utilizing parallel computing resources.

10. KNN algorithm

The¬†K-Nearest Neighbors (KNN) algorithm¬†is a non-parametric, supervised learning method used for both classification and regression tasks in machine learning.

How it works:

  1. Choose K: Decide on the number of nearest neighbors to consider.
  2. Calculate Distance: For a new, unseen data point, calculate the distance to every other data point in the training set.
  3. Find Nearest Neighbors: Identify the 'K' training data points that have the smallest distances to the new data point.
  4. Make Prediction:
    • Classification: Assign the new data point to the class that is most frequent among its 'K' nearest neighbors (majority vote).
    • Regression: Predict the value of the new data point as the average (or weighted average) of the values of its 'K' nearest neighbors.

Advantages of KNN Algorithm:

  • It is simple to implement.
  • It is robust to the noisy training data
  • It can be more effective if the training data is large.

Disadvantages of KNN Algorithm:

  • Always needs to determine the value of K which may be complex some time.

  • The computation cost is high because of calculating the distance between the data points for all the training samples.

### MapReduce Pseudo Code for Matrix Multiplication

Let's say we want to multiply two matrices, A (of size m√ón) and B (of size n√óp), resulting in a matrix C (of size m√óp).

**Map Function:**

MAP(matrix_name, row_index, column_index, value): IF matrix_name == A:

FOR k FROM 1 TO p:
  EMIT ( (row_index, k), (matrix_name, column_index, value) )

ELSE IF matrix_name == B:

FOR i FROM 1 TO m:
  EMIT ( (i, column_index), (matrix_name, row_index, value) )


**Reduce Function:**

REDUCE(key (row, col), list of values [(matrix, index, value)]): Initialize product = 0 Initialize a_values = an empty list Initialize b_values = an empty list

FOR each (matrix, index, value) in list of values:

IF matrix == A:
  Add (index, value) to a_values
ELSE IF matrix == B:
  Add (index, value) to b_values

Sort a_values by index Sort b_values by index

FOR i FROM 1 TO length(a_values):

product = product + a_values[i].value * b_values[i].value

EMIT (key (row, col), product)
