Cryptographic Algorithms

Digital Signature

Mathematical schemes that demonstrate the authenticity and integrity of document.

Example: Alice uses her private key to create a digital signature for a document. Bob receives the document and verifies the signature using Alice's public key, confirming that Alice sent the document and it hasn't been altered.

flowchart TB
	s([Signer]) --> Data --> e[Private Key
	Encryption] --> dsd[Digital Signed 
	Document] --> h([Hash]) --> d[Private Key
	Decryption] --> hv["Hash Value
	(Signature is valid if hash values match)"] --> r([Reciver])

Digital Certificates

Digital certificates are electronic credentials that verify the identity of entities (people, organizations, or devices) online.

Issued by trusted CA, they‚Äôre used in HTTPS, email encryption, and VPNs.

Example: A website obtains an SSL certificate from a CA. This certificate contains the website public key and identity information, allowing user browser to verify the website authenticity and establish a secure connection.

Public Key Infrastructure (PKI)

flowchart LR
	c([Client]) --> PublicKey -- Cipher Text --> PrivateKey --> s([Server])

It's a framework for managing digital certificates and public-private key pairs.

Example A company uses a PKI to issue digital certificates to its employees for secure email communication and access to internal resources.
