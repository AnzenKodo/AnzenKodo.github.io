CAP Theorem

  • The CAP theorem states that it is impossible for a distributed system to simultaneously provide all three of the following guarantees:
    • Consistency: Every read receives the most recent write or an error
      • RDBMS
    • Availability: Every request receives a response, without the guarantee that it contains the most recent write
      • MongoDB, HBase
    • Partition Tolerance: system continues to operate despite arbitrary partitioning due to network failures
      • Cassandra, CouchDB

BASE Property

  • Basically Available
    • If one node fails, other nodes can still serve read and write requests.
  • Soft State
    • Data changes might take time to be replicated across all nodes.
    • During this time, different clients might see different versions of the data.
    • The system is in a state of flux ("soft") until consistency is achieved. ¬†
  • Eventually Consistent

CURE

  • Full name Clustering Using Representatives
  • Clustering is classification of objects into groups.
  • Works with outliers.
  • Efficient with large datasets.
  • It is hierarchical algorithm, it starts from single point and merges clusters until number is reached.

  • CURE Architecture:

    flowchart TB
	0[(Data)] -->
	1[Draw Random Sample] -->
	2[Partion Sample] -->
	3[Partially Cluster's
	Partitions] -->
	4[Eliminate Outliers] -->
	5[Cluster Partial Clusters] -->
	6[Label Disk Data] -->
	7[(Clusters)]

  • Partition Sample: The random sample is divided into smaller partitions or groups

  • Partially Cluster's Partitions: Clustering is applied to each of the individual partitions

  • Cluster Partial Cluster: The results from the partially clustered and outlier-cleaned partitions are combined and clustered again

Advantages over traditional clustering algorithm

  • Handles Non-Spherical Shapes
  • Robust to Outliers
  • Scalability for Large Datasets
  • Better Cluster Boundary Recognition: Using multiple representative points helps CURE to better recognize and define the boundaries between clusters

DGIM algorithm

(10 Marks, 10 Marks, 10 Marks, 5 Marks) (2024 December, 2024 May, 2023 December, 2023 May)

he Datar-Gionis-Indyk-Motwani (DGIM) Algorithm is used to estimate the number of 1's in a stream within a window of N bits.

Rules

  1. The right end of a bucket is always a position with a 1.
  2. Every position with a 1 is in some bucket.
  3. No position (1's, 0's) can be in more then one bucket.
  4. There are one or two buckets of any given size (which must be a power of 2).
  5. All buckets should be power of 2.
  6. Buckets cannot decrease in size as we move to the left (back in time).

$$\text{No of 1's: } 2^2 + 2^2 + 2^1 = 4 + 4 + 2 = 10 $$ There are 10 bits of 1's in the given dataset.

It reduces the size of candidate sets in the A-Priori algorithm by utilizing unused space in main memory.

  • Bitmap: the count of pairs that hash to a particular bucket exceeds a certain threshold (support count), the corresponding bit in the bitmap is set to 1; otherwise, it's set to 0.

Support Vector Machine (SVM)

SVM stands for , which is a supervised machine learning algorithm used for classification and regression tasks.

SVMs can be used for a variety of tasks, such as text classification, image classification, spam detection,¬†handwriting identification, gene expression analysis, face detection, and anomaly detection

Types of Support Vector Machine

  • Linear SVM:¬†
    • Linear SVMs use a linear decision boundary to separate the data points of different classes.
  • Non-Linear SVM:¬†
    • Non-Linear SVM can be used to classify data when it cannot be separated into two classes by a straight line (in the case of 2D).

Support Vector Machine Terminology

  1. Data Points
  2. Hyperplane A hyperplane is like a decision boundary that separates different classes in the data.
  3. Positive Hyperplane and Negative Hyperplane: The positive hyperplane passes through the support vectors of the positive class, and the negative hyperplane passes through the support vectors of the negative class.
  4. Support Vectors: Support vectors are the data points closest to the hyperplane.
  5. Maximum Margin: The margin is the distance between the hyperplane and the closest data points.
