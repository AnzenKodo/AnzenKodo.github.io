Database as a Service Vs Storage as a Service

Feature Database as a Service (DBaaS) Storage as a Service (STaaS)
Definition CSP managed database systems CSP managed data storage
Primary Function Manages, stores, and processes structured data Stores, retrieves, and manages files or objects
User Management Handles database provisioning, scaling, backups, security Handles storage provisioning, scaling, redundancy
Data Type Structured data Unstructured/structured data
Data tables, queries, transactions files, objects, backups
Use Cases Application data, analytics, transactional processing File storage, backup, archiving, disaster recovery
Access Method Accessed via database protocols Accessed via file/object storage protocols
Applications NoSQL, MySQL S3, NFS

Identity and Access Management (IAM)

  • It is a framework comprising policies, processes and technologies that manages digital identities and control access.
  • Key components
    • Identity Management
    • Authentication
    • Authorization
    • Access Control

  • Standards and Protocols

    • OAuth 2.0: framework that enables applications to obtain access to accounts through HTTP.
    • OpenID Connect (OIDC):
      • An identity layer build on top of OAuth 2.0
      • Used for verifying user identity.

    • Kerberos

      • Authentication protocol that provides security through secret-key cryptography.

Virtualization

Xen

  • Type-1 or Bare Metal hypervisor
  • MMU (Memory Management Unit)
  • How it is Helpful in Large Scale Virtualization
    • Performance
    • Scalability
    • Security
    • Resource Control
  • Compontes
    • VM0 (or Dom0)
      • First VM.
      • Has special privileges
      • Directory interact with Xen and Hardware
      • Runs System services (TS, DE, XS)
    • VM1 TO VMn:
      • Guest VM runs own Guest OS and Apps.
      • Managed by Xen and Dom0

Xen Vs KVM

Feature Xen Architecture KVM Architecture
Type Type 1 (bare-metal) Type 2 (hosted, in Linux kernel)
Placement Runs below OS, directly on hardware Runs as part of Linux kernel
Resources Usage Low High
Kernal Type Micro Kernal Linux Kernal
Security Model Strong Isolation Linux Kernal Securtiy
Guest OS Support Paravirtualizatoin & Hardware-assisted Full Virtualization (unmodified OS)
Allocation CPU Cores Not Supported Supported
Memory Management Dynamic Traditional Mapping per VM

Paravirtualization: where the guest OS is aware it's running in a virtual environment. Hardware-assisted: utilizes hardware features (primarily in the CPU) to improve virtualization performance.

Elastic Block Storage (EBS)

  • block level storage
  • SSD & HDD
  • Persistent storage
  • Independent of the instance lifecycle

Simple Storage Service (S3)

  • Stores data as objects within buckets
  • Advantages
    • Scalability
    • Cost Effective
    • Security
  • Disadvantages
    • Complexity
    • Cost Management
    • Object Limit

Glacier

  • Provides low-cost archival storage for long-term data retention.
  • Designed for infrequently accessed data.
  • Why it is required?:
    • Long-term data preservation
    • Low-cost
    • High durability
    • Security

DynamoDB

flowchart LR
	API --> DynamoDB --> Table1 & Table2 & Table3
  • It a NoSQL database
  • Data Structure
    • Tables: These are similar to tables in relational databases but don't have a fixed schema.
    • Items: These are analogous to rows or records in relational databases. Each item is uniquely identified by a primary key.
  • Features
    • Fully Managed
    • High performance
    • Scalable
    • Serverless
    • Flexible data model
      • Schema-less, ideal for unstructured data
      • Supports key-value and document models.

  • MCC is combination of Mobile Computing(MC), Cloud Computing (CC) & Wireless Network (WN).
  • It makes it possible for reach mobile application to be executed on huge number of devices.
  • Components
    • BTS (Base Station)
    • AAA (Authorization, Authentication, Accounting)
    • Home Agent (HA): maintains info about devices current location.
  • Benefits
    • Flexibility
    • Multiple Platform Support
    • Backup & Recovery
  • Challenges
    • Attacks & Hacking
    • Low bandwidth problem
    • Information Security Privacy

Governance, Risk, and Compliance (GRC)

It's structured approach that integrates an organization's governance, risk management, and compliance activities.

It aims to align IT with business goals, manage risks effectively, and adhere to relevant laws and regulations.

Components

  • Governance: Establish the org structure, policies, and processes for decision-making and accountability.
  • Risk Management
  • Compliance

Cloud Cube Model

  • Internal: Data inside the org.
  • External: Data stored in outside the org (Exp: AWS & Google Cloud)
  • Proprietary: Owned by CSP with restricted interoperability or locked-in ecosystem.
  • Open: The service use open standard, which allows greater interoperability, data portability and flexibility.
  • Permiterized: Operations occur within traditional security boundaries.
  • De-permiterized: Security is designed for boundary-less enviorment.
