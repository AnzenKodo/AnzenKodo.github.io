Addressing modes of the 8086 microprocessors (2)

https://youtu.be/R4GCO5wKDkQ What is Addressing Mode

  • Immediate Addressing Mode

    • In this mode the operand is specified in instruction itself.

    • Instruction are longer but the operand are easily identified.

      .asm 
      .text
main:
mov cl, 12H
mov bx, 1234H

  • Register Addressing Mode

    • Operands is save on register and it move other register to perform operation.
    • In this mode operands are specified using Registers.

    • Instruction are shorter but operands can't be identified by looking at instruction.

      .asm 
      .text
main:
mov cl, dl
mov ax, bx

  • Direct Addressing Mode

    • In instruction you get memory location.

    • In this mode address of operand is directly specified in the instruction.

      .asm 
      .text
main:
mov cl, [4321h] 
; Moves data from 4321h in the data segment into `cl`.
; The physical address is calculated as `ds*10n+4521` = `5000x10n+4321h`
mov cx, [4320h]

  • Indirect Addressing Mode

    • Register Indirect Addressing Mode

      • In this mode the up uses any of the 2 base registers BP, BX or any of the two index registers SI, Di to provide the offset address for the data type.

        .asm 
        .text
main:
mov cl, [bx]

    • Register Relative Addressing Mode

      • In this mode the operand address is calculated using one of the base registers and a 8-bit or a 16-bit displacement.

        .asm 
        .text
main:
mov cl [bx+4]
; moves a byte from address pointed by `[bx+4]` in data seg to `cl`

    • Base Indexed Addressing Mode

      • Directly source index and destination index will be added.

      • Operand address is calculated as base register plus and index register.

        .asm 
        .text
main:
mov cl, [bx+si]
; `si` is sorce index
; moves a byte from the address pointed by bx+si in data segment to cl.
; physical address `ds+10h+bx+si`

    • Base Relative Plus Index Addressing Mode

      • In this mode the address of the operand is calculated as base register plus index register plus 8-bit or 16-bit displacement.

        .asm 
        .text
main:
mov cl, [bx-di+20]

    • Implied Addressing Mode

      • In this addressing mode the operands are implied and are not specified in the instruction.

        .text
main:
stc ; sets the carry flag
cld ; decales the direction flag

Arithmetic instructions of 8086

  • ADD (Addition)

    • Adds the source to the destination and stores the result back in the destination.
    • Source: Register, Memory location, Immediate number

    • Destination: Register

      .asm 
      ADD AL, 25H ; AL <- AL+25H

  • ADC (Addition with Carry)

    • It adds the carry into the result.

      .asm 
      ADC BX, CX ; BX <- BX+CX+Carry Flag

  • SUB (Subtraction)

    • It is similar to ADD except that it does subtraction.
    • Subtracts the source from the destination and stores the result back in the destination.
    • Source: Register, Memory location, Immediate number

    • Destination: Register

      .asm 
      SUB AL, 25H ; AL <- AL-25

  • SBB (Subtraction with Borrow)

    • It is similar to ADC except that it does subtraction.

    • It subtracts with the borrow.

      .asm 
      SBB BL, CL ; BL <- BL-CL

  • INC (Increment)

    • Adds "1" to the specified destination.
    • Destination: Register, Memory Location.

    • Note: Carry flag is not affected.

      .asm 
      INC AX ; AX <- AX + 1

  • DEC (Decrement)

    • Subtracts "1" from the specified destination.
    • Destination: Register, Memory Location.

    • Note: Carry flag is not affected.

      .asm 
      DEC BL ; BL <- BL - 1

  • MUL (Multiplication)

    • Multiplies an 8-bit or 16-bit register with the accumulator.

    • The result is stored in the accumulator (AX) or in DX:AX for 16-bit multiplication.

      .asm 
      MUL BH ; AX <- AX * BH

  • IMUL (Signed Multiplication)

    • Performs signed multiplication of an 8-bit or 16-bit register with the accumulator.

      .asm 
      IMUL CX ; AX <- AX * CX (signed multiplication)

  • DIV (Division)

    • Divides the accumulator by an 8-bit or 16-bit register.

    • Quotient is stored in the accumulator (AL for 8-bit, AX for 16-bit) and remainder in AH (for 8-bit) or DX (for 16-bit).

      .asm 
      DIV BL ; AL <- AX / BL, AH <- Remainder

  • IDIV (Signed Division)

    • Performs signed division of the accumulator by an 8-bit or 16-bit register.

      .asm 
      IDIV BL ; AL <- AX / BL (signed division), AH <- Remainder

  • NEG (Negation)

    • Replaces the value of the specified destination with its two's complement.

      .asm 
      NEG AL ; AL <- 2’s complement of AL

  • CBW (Convert Byte to Word)

    • Converts a signed byte in AL to a signed word in AX.

      .asm 
      CBW ; AL -> AX (sign extended)

  • CWD (Convert Word to Double Word)

    • Converts a signed word in AX to a signed double word in DX:AX.

      .asm 
      CWD ; AX -> DX:AX (sign extended)

  • DAA, DAS, AAA, AAS, AAM, AAD

    • These are BCD (Binary Coded Decimal) arithmetic instructions used for BCD operations.

Direct Memory Access (2)

Enables data transfer between I/O and memory without CPU intervention.

Need a hardware: DMAC (DMA Controller)

DMAC is a special purpose processer, which controls data transfer b/w memory and I/O (because it generates add & controls signals for memory)

DMA could work even when instruction is executing.

Starting Address: Memory address. starting from where data transfer should be performed.

Data Count: No. of bytes of words to be transferred.

Initially After 1B After 2B After 3B
SA 1000 1001 1002 1003
DC 500 499 498 497

During DMA transfer CPU can perform only those operations which do not require system but which means mostly CPU will be blocked.

Whenever control is given to DMAC then do not give it for a larger time.

To find, how much time System Bus is placed with DMAC we use Modes of DMA.

Modes of DMA

  • Burst Mode
    • Burst of data is transferred before CPU takes the control of bases back
    • Bust of data could be:
      • Entire Data
      • Bust of blocks
  • Cycle Stealing
    • Slow I/O device takes some time to prepare the word.
    • During this time CPU keeps control of the buses.
    • Once word is ready CPU gives the control of the buses to DMAC for cycle in which prepare word is transferred to memory.

  • Interleaving DMA

    • Whenever CPU does not require system buses (doing internal work) the only control of the buses will be given to DMAC.
    • CPU will not be blocked due to DMA
    • Maximum time required for data transfer as compared to burst & cycle stealing mode.
    • Time required for data transfer: Interleaving > Cycle Stealing > Burst Mode

    • Speed of data transfer: Bust Mode > Cycle Stealing > Interleaving

Types of Memory

Primary Memory

  • Is a essential component of a computer system.

    • Program and data are loaded into the primary memory before processing.
    • CPU interacts directly with the primary memory to perform read or write operations.
    • There are two types of primary memory:
    • RAM (Random Access Memory)
      • RAM is volatile memory that temporarily stores the files you are working on.
      • Each time you switch from one program or page to another, the information is instantly available.
      • When the computer is shut-down, the memory is cleared until the process begins again.
    • ROM (Read-Only Memory)
      • ROM is non-volatile memory, which means the information is permanently stored on the chip.
      • Non-volatile memory is used for parts of the computer that don not change, such as the initial boot-up portion of the software.
      • Turing off the computer does not any effect on ROM.
      • Non-volatile memory can't be changed by users.

    • Also can be called DRAM

      Cache Memory

  • Cache memory is a special very high speed memory.

  • It is used to speed up and synchronize with high-speed CPU.

  • Cache memory is an extremely fast memory type that acts as a buffer between RAM and CPU.

  • It reduces the average time to access data from the Main memory.

  • Cache memory is closer then main memory to the CPU.

  • Also can be called SRAM

    Secondary Memory

  • Primary memory has limited storage capacity.

  • So a computer system needs auxiliary or secondary to permanently store the data or instruction for future use.

  • The secondary memory is non-volatile and has a larger storage capacity the primary memory.

  • It is slower and cheaper the main memory.

  • It can't be accessed directly by the CPU.

  • Contents of secondary storage need to first brought into the main memory for the CPU to access.

Data Transfer between Memory and CPU

  • Data need to be transferred between CPU and Primary memory as well as between the primary and secondary memory.
  • Data is transferred between different components of a computer system using physical wires called the bus.
  • There are 3 types of buses:
    1. Data Bus
    2. Address Bus
    3. Control Bus

Pipeline Hazards

Hazard means Problem.

CPI(Clock Per Instruction) should be 1 in pipeline but achieving it difficult.

To achieve the CPI = 1 it creates problem and it is called Pipeline Hazards.

There are three types of hazards in pipeline:

Data Hazards

  • RAW(TD) - Read After Write (True Dependency)
    • Pipeline reading instruction after writing.
  • WAR(AD) - Write After Read (Added Dependency)
  • WAW(OD) - Write After Write (Output Dependency)

Structural Hazards

  • Multiple instruction trying to access single resources.
  • Resources like: Memory, Registers, RAM, Cache and many other.

Control Hazards

  • It occurs because of branch conditions.
